# App Audit — 2026-03-18
## Speed, UX, and Customization Review

### 30 Findings (sorted by severity)

---

### CRITICAL (5) — User would notice immediately

**1. Dashboard auto-generates AI insight on every visit**
- File: Dashboard.jsx ~line 365
- Category: Speed / Cost
- Issue: `generateInsights()` auto-runs on first load. Wastes an AI call every time you navigate to Dashboard if cache is missing or stale.
- Fix: Only auto-generate if no cache exists. Show a "Generate insights" button for refresh.

**2. Tasks page makes 3 sequential storage fetches**
- File: Tasks.jsx ~line 78-83
- Category: Speed
- Issue: `getTaskItems()`, `getAutopilotActions()`, `getAccounts()`, `getSettings()` all run sequentially. Adds 2-4 seconds to page load.
- Fix: Use `Promise.all()` to fetch in parallel.

**3. All agents use serial context loading loops**
- Files: AgentHub.jsx, ExpansionScout, RescuePlanner, OutreachDrafter, etc.
- Category: Speed
- Issue: `for (const a of accounts) { await renewalStore.getContext(a.id) }` — serial loop. With 20 accounts, this adds 5-10 seconds.
- Fix: `await Promise.all(accounts.map(a => renewalStore.getContext(a.id)))` — parallel.

**4. Dashboard shows no loading state during initial account fetch**
- File: Dashboard.jsx ~line 21
- Category: UX / Bug
- Issue: `renewalStore.getAccounts().then(setAccounts)` has no loading state. User sees brief blank flash.
- Fix: Add loading spinner or skeleton before accounts resolve.

**5. ExpansionScout double-fetches context data**
- File: ExpansionScout.jsx ~line 60-103
- Category: Speed / Duplication
- Issue: Health results are computed (fetching context), then `analyzeExpansion()` fetches context AGAIN for the same accounts.
- Fix: Cache context from initial load and pass it to the analysis function.
